# US-19 — Créer et éditer son profil utilisateur

**Récit**
En tant qu'utilisateur de Bary, je veux définir mon prénom et ma station de départ lors de ma première session, puis pouvoir les modifier à tout moment depuis le carnet, afin d'être pré-ajouté automatiquement à mes groupes sans ressaisie.

**Priorité** : Must
**Statut** : Livré

---

### Critères d'acceptation

**Scénario 1 — Onboarding différé : création du profil au premier groupe**
- **Given** l'utilisateur n'a pas encore de profil enregistré (localStorage vide)
- **When** il clique "Créer un point de rencontre" sur l'écran d'accueil
- **Then** une modale légère s'affiche avec le message *"Et toi, tu pars d'où ?"*, un champ prénom et un autocomplete station ; deux actions sont disponibles : **"C'est parti"** (sauvegarde le profil et ouvre l'écran Groupe avec l'utilisateur pré-ajouté) ou **"Continuer sans profil"** (ferme la modale et ouvre l'écran Groupe vide, sans pré-ajout)

**Scénario 2 — Sauvegarde du profil**
- **Given** l'utilisateur a saisi un prénom valide et sélectionné une station dans l'autocomplete
- **When** il tape "C'est parti"
- **Then** le profil est persisté en localStorage (`bary_profile`) ; une entrée correspondante est créée dans le carnet d'amis avec le flag `isMe: true` ; l'application ouvre l'écran Groupe avec l'utilisateur pré-ajouté comme premier participant

**Scénario 3 — Accès et modification du profil via le carnet**
- **Given** l'utilisateur a un profil enregistré et ouvre l'écran Carnet d'amis
- **When** il consulte la liste
- **Then** sa fiche *"Moi"* apparaît en première position (épinglée), avec un badge distinctif "Moi", un bouton **Modifier** (comme les autres amis) et **sans bouton ×** (non supprimable) ; en tapant Modifier, il peut changer son prénom ou sa station par défaut ; les nouvelles valeurs s'appliquent aux **prochaines sessions uniquement** — la session en cours n'est pas affectée

**Scénario 4 — Tentative de suppression de son propre profil dans le carnet**
- **Given** l'utilisateur consulte le carnet d'amis
- **When** il tente de supprimer l'entrée marquée `isMe: true`
- **Then** l'action est impossible — aucun bouton × n'est affiché pour cette entrée ; l'entrée reste dans le carnet

---

### Dépendances
- `US-20` — le profil est utilisé pour le pré-ajout automatique à chaque nouvelle session
- `US-21` — la station peut être modifiée localement depuis l'écran Groupe (sans affecter le profil par défaut)

### Edge cases identifiés
- **Prénom vide ou espaces uniquement** : validation inline, bouton "C'est parti" désactivé
- **Caractères interdits dans le prénom** (`, | & = + # ? %`) : même règle que l'ajout d'un ami (voir BUG-03, BUG-04)
- **Utilisateur ignore la modale d'onboarding** : pas de profil créé, le groupe démarre vide — la modale peut réapparaître à la prochaine session si toujours aucun profil
- **localStorage indisponible** (navigation privée, quota dépassé) : afficher un message d'avertissement ; continuer en session uniquement sans persistance
- **Modification du profil alors qu'une session est en cours** : les nouvelles valeurs ne s'appliquent pas à la session en cours — uniquement aux prochaines sessions

### Notes techniques / Contraintes connues
- Profil stocké sous la clé `bary_profile` en localStorage
- Le carnet (`bary_friends`) contient une entrée `isMe: true` synchronisée avec le profil — les deux sont mis à jour de façon atomique
- Le flag `isMe: true` est la source de vérité pour : (1) masquer le bouton × dans le carnet, (2) épingler l'entrée en tête de liste, (3) afficher le badge "Moi"
- La modale d'onboarding s'affiche uniquement si `bary_profile` est absent de localStorage au moment du clic "Créer un point de rencontre"

### Lien maquette
- Modale "Et toi, tu pars d'où ?" (onboarding différé, premier groupe)
- Écran Carnet d'amis (fiche "Moi" épinglée, badge, sans bouton ×)

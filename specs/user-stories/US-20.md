# US-20 — Être pré-ajouté automatiquement comme premier participant

**Récit**
En tant qu'utilisateur ayant configuré mon profil, je veux être automatiquement présent comme premier participant à chaque nouvelle session, afin de ne pas avoir à saisir ma station à chaque fois.

**Priorité** : Must
**Statut** : Draft

---

### Critères d'acceptation

**Scénario 1 — Ouverture d'une nouvelle session avec profil existant**
- **Given** l'utilisateur a un profil enregistré (prénom + station en localStorage)
- **When** il clique "Créer un point de rencontre" et passe la modale d'onboarding (ou elle n'est pas affichée car le profil existe déjà)
- **Then** l'écran Groupe s'ouvre avec son entrée déjà présente en première position, marquée visuellement avec un badge "Moi" ; il possède un bouton de retrait comme n'importe quel participant

**Scénario 2 — Retrait de soi-même du groupe et ré-ajout**
- **Given** l'utilisateur est présent dans le groupe comme premier participant
- **When** il tape le bouton de retrait sur sa propre entrée
- **Then** son entrée disparaît de la liste des participants de la session en cours ; pour se ré-ajouter, il passe par le carnet d'amis — sa fiche "Moi" y est toujours présente (non supprimable, voir US-19) ; la modification de sa station dans le groupe passe par US-21 et reste locale à la session

**Scénario 3 — Ouverture d'une nouvelle session sans profil**
- **Given** l'utilisateur n'a pas de profil (a ignoré la modale d'onboarding ou n'a jamais cliqué "Créer un point de rencontre" avant)
- **When** il accède à l'écran Groupe
- **Then** le groupe démarre vide — aucun participant pré-ajouté ; l'utilisateur ajoute les participants manuellement

---

### Dépendances
- `US-19` — le profil doit exister pour que le pré-ajout soit possible ; US-19 définit aussi la règle de non-suppression de l'entrée `isMe: true` dans le carnet, qui permet le ré-ajout après retrait
- `US-21` — pour modifier la station du participant pré-ajouté directement depuis l'écran Groupe sans affecter le profil par défaut

### Edge cases identifiés
- **Profil corrompu ou incomplet en localStorage** (prénom présent mais station absente) : l'utilisateur n'est pas pré-ajouté ; la modale d'onboarding "Et toi, tu pars d'où ?" est proposée au prochain clic "Créer un point de rencontre"
- **Session restaurée depuis localStorage avec le participant "moi" déjà présent** : ne pas dupliquer l'entrée au rechargement de la page
- **Modification du profil (US-19) alors qu'une session est en cours** : la session n'est pas affectée — les nouvelles valeurs s'appliquent à la prochaine session uniquement

### Notes techniques / Contraintes connues
- Le pré-ajout lit le profil depuis localStorage (`bary_profile`) à l'initialisation de la session ; il ne dépend pas d'un état React global non persisté
- L'entrée "moi" dans la liste des participants est identifiable par un flag `isMe: true` qui conditionne le badge visuel et l'ordre (toujours en tête de liste)
- Le retrait de "moi" du groupe ne supprime pas l'entrée du carnet d'amis ; il retire simplement le participant de la session en cours

- Écran Carnet d'amis (fiche "Moi" disponible pour ré-ajout après retrait)

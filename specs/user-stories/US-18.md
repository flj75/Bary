# US-18 — Ouvrir un lien partagé et afficher le résultat

**Récit**
En tant que destinataire d'un lien Bary, je veux qu'en ouvrant le lien dans mon navigateur je voie directement l'écran 4 avec le résultat pré-calculé, afin d'accéder au point de rencontre sans refaire le calcul.

**Priorité** : Should
**Statut** : Bloqué

---

### Critères d'acceptation

**Scénario 1 — Ouverture du lien partagé**
- **Given** un destinataire reçoit le lien `bary.app/result?s=[station-id]&g=[prenom|station-id,...]`
- **When** il ouvre le lien dans son navigateur
- **Then** il voit directement l'écran 4 avec la station optimale, les participants et leurs temps de trajet

**Scénario 2 — Lien malformé ou station introuvable**
- **Given** le lien contient un stationId invalide ou manquant
- **When** la page se charge
- **Then** l'utilisateur est redirigé vers l'Écran 1 avec un message "Lien invalide"

---

### Dépendances
- `US-10` — l'émetteur génère le lien
- **Bloqué par `BUG-01`** — le parseur doit utiliser `|` comme séparateur nom/stationId (voir `specs/bugs/BUG-01.md`)

### Edge cases identifiés
- Lien malformé ou station introuvable → redirection Écran 1 + message "Lien invalide"
- StationId IDFM contenant `:` → résolu par le correctif BUG-01 (séparateur `|`)
- Participant avec station introuvable dans le dataset → ignorer ou bloquer (à définir)

### Notes techniques / Contraintes connues
- **Bloqué par BUG-01** : `URLSearchParams.get('g')` décode `%3A` → `:` avant retour. Un `split(':')` naïf sur `IDFM:463079` produit 3 parties au lieu de 2. Correctif documenté dans `specs/bugs/BUG-01.md` : utiliser `|` comme séparateur.
- Format URL cible : `bary.app/result?s=[station-id]&g=[prenom|station-id,prenom|station-id,...]`
- La page `/result` doit détecter l'absence de session (`result=null`) ET la présence de params URL, et reconstituer le résultat à partir des params côté client uniquement (pas de backend)
- Le `buildShareUrl` dans `src/app/result/page.tsx` (ligne 39, TODO marqué) doit passer de `:` à `|` comme séparateur

### Lien maquette
- Écran 4 — Résultat (vue destinataire)

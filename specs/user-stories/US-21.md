# US-21 — Modifier la station d'un participant en cours de session

**Récit**
En tant qu'utilisateur en cours de session, je veux pouvoir modifier la station d'un participant déjà ajouté sans le supprimer et le ré-ajouter, afin de corriger une erreur ou mettre à jour son point de départ rapidement.

**Priorité** : Should
**Statut** : Draft

---

### Critères d'acceptation

**Scénario 1 — Édition inline de la station d'un participant**
- **Given** la liste des participants est affichée (Écran 2)
- **When** l'utilisateur tape sur la station d'un participant (ou sur une icône d'édition associée)
- **Then** un champ de saisie avec autocomplete IDFM s'ouvre sur cette ligne ; la station actuelle est pré-remplie dans le champ ; les autres participants restent visibles

**Scénario 2 — Validation de la nouvelle station**
- **Given** l'utilisateur a sélectionné une nouvelle station dans l'autocomplete
- **When** il confirme la sélection (tap sur un résultat de l'autocomplete ou validation du champ)
- **Then** la station du participant est mise à jour dans la session uniquement ; le champ d'édition se referme ; la liste affiche la nouvelle station

**Scénario 3 — Annulation de l'édition**
- **Given** le champ d'édition est ouvert
- **When** l'utilisateur ferme le champ sans sélectionner de station (tap en dehors, bouton Annuler, ou touche Echap)
- **Then** la station du participant reste inchangée ; le champ d'édition se referme sans modification

---

### Dépendances
- `US-20` — US-21 est le mécanisme d'édition référencé dans US-20 (Scénario 2) pour modifier la station du participant "moi" depuis la liste du groupe

### Edge cases identifiés
- **Champ vidé sans nouvelle sélection** : la validation se fait par auto-commit à la sélection d'un résultat — si l'utilisateur vide le champ sans sélectionner, l'édition reste ouverte jusqu'à Cancel ou Escape ; aucun message bloquant n'est nécessaire
- **Édition de la station de "moi"** : même comportement que pour n'importe quel participant — la modification est locale à la session en cours et ne met pas à jour le profil par défaut en localStorage
- **Deux participants avec la même station après modification** : comportement identique au cas normal (résultat valide, temps = 0 pour les deux)

### Notes techniques / Contraintes connues
- L'autocomplete utilisé est le même composant `StationAutocomplete` que pour l'ajout d'un participant — réutilisation directe, pas de nouveau composant
- La modification est locale à la session en cours ; elle ne modifie ni le profil en localStorage ni le carnet d'amis
